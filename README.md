# B站评论盖楼 (Bili-comment_stacker)

用于在指定的 Bilibili 视频评论区，自动抓取最新的评论进度（如中V打卡），并按设定的时间间隔，自动发送递增的序列评论。

我很难保证每天都有时间去刷评论盖楼，就想出了这样一个偷懒工具，挂在服务器上，基于公开API，模拟用户操作。

# ✨ 核心功能

- 智能进度识别：自动抓取最新 10 条评论，正则匹配并提取当前最大进度（如 711）。

- 自动递增发送：计算 +1（如 712），拼接完整文案并发送。

- 进度上限控制：达到设定的最大打卡次数后，脚本自动安全终止。

- 本地日志留档：运行状态与接口返回信息将追加记录至本地 comment_record.txt。

#  环境与依赖

Python 3.x

仅需安装 requests 库：

```Bash
pip install -r requirements.txt
```

# ⚙️ 配置说明
运行前，请使用文本编辑器打开 1.py，并在文件顶部的 “用户配置区” 填写参数：

```Python
# 目标视频的 BV 号
BVID = "BV1Y26wBcERw"

# 账号鉴权信息 (浏览器 F12 抓包获取)
SESSDATA = "替换为你的SESSDATA"
CSRF = "替换为你的bili_jct" 

# 评论模板 (程序会自动在 PREFIX 和 SUFFIX 之间插入递增的数字)
PREFIX = "唤炽心无双，（"
SUFFIX = "/1000）秋同所向，一弦一调唤知己，山海风流鸣笙簧，乐鸣东方！"

# 循环间隔时间 (小时)
INTERVAL_HOURS = 24 

# 进度上限
MAX_LIMIT = 1000
```

🔑 **关于鉴权获取：**

网页端登录 B站 ->  F12  -> Application (应用) -> Cookies -> 复制 SESSDATA 和 bili_jct 的值。（**请注意保护 Cookie 隐私，切勿外传**）。

# 🚀 运行脚本

```
python 1.py
```

运行后脚本会首先获取并确认视频信息，随后进入自动轮询状态。

# ⚠️ 注意事项
1. 频繁的自动化评论会触发B站的风控机制（如禁言、封号、评论仅自己可见）,也不利于社区风气。请合理设置 INTERVAL_HOURS（建议不要低于 24 小时）。

2. 遵守社区规则：请确保发送的评论内容符合 Bilibili 社区规范，避免发送垃圾广告或恶意刷屏。
